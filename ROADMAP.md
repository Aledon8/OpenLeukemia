# OpenLeukemia Roadmap

The project is being built in stages.

Not every idea belongs in the first release.

Some ideas are long-term research directions.

---

# Phase 0 — Foundation (current)

Goal:

Build a simple, privacy-first patient platform.

Features:

- user accounts
- consent system
- Supabase infrastructure
- PDF upload
- manual lab entry
- patient timeline
- document extraction
- basic explanations
- export and delete data

Principles:

- patient owns data
- no broad consent
- AI assists, not diagnoses

---

# Phase 1 — Better understanding

Goals:

Reduce confusion around medical information.

Possible features:

- trend visualization
- medical terminology explanations
- document comparison
- structured history
- FAQ assistant

---

# Phase 2 — Privacy-first AI

Goals:

Move sensitive processing closer to the patient.

Possible exploration:

- local AI workflows
- device-side processing
- selective offline features
- local embeddings

Note:

Not all devices can support local processing.

Cloud alternatives will remain available.

---

# Phase 3 — Research directions

Long-term ideas:

- federated learning
- privacy-preserving model updates
- secure aggregation
- collaborative model improvement

These ideas are experimental and are not required for MVP.

---

# Community-driven direction

OpenLeukemia is not intended to grow behind closed decisions.

Patients, developers, researchers, designers, and community members can help shape the future direction of the project.

Community ideas, feature proposals, and workflow suggestions are welcome through GitHub Discussions.

Possible examples:

- patient experience improvements
- new tools
- accessibility ideas
- AI features
- privacy suggestions
- workflow improvements

Every proposal will be reviewed and discussed.

Not every idea will become part of the project, but every idea deserves consideration.

The roadmap is intended to evolve together with the community.
