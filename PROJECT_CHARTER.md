# OpenLeukemia / Project Remission

## Project Charter and Community Policy

Version: 0.1  
Status: Draft

---

# Our Mission

This project is built by patients, developers, researchers, and people who care about the lives of people living with cancer.

Our goal is to create tools that help people better understand their condition, simplify working with medical data, reduce anxiety, and save time.

We are not building a project around illness.  
We are building a project around the person.

The project is not created to maximize profit.  
Project revenue exists to support:

* infrastructure;
* server costs;
* software development;
* research;
* community support;
* patient assistance;
* funding further development.

---

# Core Principles

## 1. The patient owns their data

Patient data belongs only to the patient.

The project is not the owner of:

* lab results;
* medical records;
* symptoms;
* images;
* documents;
* medical history.

The project only provides tools for storage and processing.

The patient can:

* download their data;
* delete their data;
* turn off synchronization;
* use the system entirely locally.

---

## 2. Privacy by default

Security and confidentiality are mandatory parts of the architecture.

By default:

* data is encrypted;
* transmission is protected;
* identity is separated from medical data;
* data collection is minimized.

We do not collect data that the project does not need.

---

## 3. We do not sell data

Never.

The project prohibits:

* selling medical data;
* sharing data with advertising companies;
* hidden use of data;
* using data for marketing;
* using data for insurance scoring;
* sharing data with third parties without consent.

---

## 4. Consent must be explicit

The user decides on their own:

[ ] store data in the cloud

[ ] participate in improving models

[ ] participate in research

[ ] share anonymized data

[ ] help train AI

Each consent can be withdrawn.

Registration does not imply blanket consent.  
Each permission must be requested separately.

---

## 5. AI assists; it does not diagnose

The project does not replace:

* a physician;
* medical consultation;
* a laboratory;
* diagnosis.

AI is intended for:

* analyzing trends;
* finding changes;
* explaining data;
* helping understand information.

The system must not state:

“You have a relapse”

The system may state:

“Changes have been detected. We recommend discussing the results with your doctor”

---

## 6. Open source by default

The project follows a semi–open-source model.

What may be open:

* code;
* documentation;
* data schemas;
* algorithms;
* APIs;
* hardware schematics;
* research.

We want developers to be able to:

* improve the project;
* build add-ons;
* propose new ideas;
* help patients.

---

## 7. Commerce must not harm people

Monetization exists for sustainability.

Monetization must not:

* limit patient access;
* turn illness into a profit center;
* reduce the quality of care;
* use fear or pressure.

When possible, part of revenue may be directed toward:

* patient assistance;
* charitable foundations;
* community development.

---

# Security Policy

The project aims to use:

* local encryption;
* end-to-end encryption;
* data minimization;
* regular audits;
* open security-related code.

Security is not built on trust.  
Security is built on verifiability.

When possible, the project should prefer privacy-preserving approaches to model improvement, including federated learning, where raw patient data remains on the patient's device and only model updates are shared with the server.

---

# Rules for Project Participants

The following are prohibited:

* using patient data outside the project;
* publishing personal information;
* implementing hidden data collection;
* creating discriminatory features;
* using the project for manipulation;
* using the project to advertise medications.
